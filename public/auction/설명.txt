모든 js 부분에서는 init() 함수만 실행됨, init 함수 내부만 따라가며 읽으면 됨

1. activeAuc.js
    auction.html에 연결되어, 네비게이션 바를 통해 옥션 페이지로 이동할 때 사용됨
    네비게이션 바에서 이동할 때 주소 뒤에 쿼리 스트링(ex. ?page=1)을 붙여 이동하게 되는데
    = 뒤의 숫자에 따라 마켓(1), 1차 옥션(2), 2차 옥션(3), 심사중인 옥션(4)의 탭을 보여주게 됨 
    (해당 탭의 classList에 active를 추가하게 됨)

2. addMarketInfo.html
    옥션의 마켓 탭에 필요한 정보를 입력할 때 사용되는 임시 페이지
    firestore의 market에 해당 상품/서비스의 데이터를 넣어줌, 사진 이미지는 imgForMarket/상품id 폴더에 저장
    이때 대/소분류 정보는 0부터 시작하는 숫자로 저장되기 때문에, 
    만일 auction.html의 대/소분류가 바뀔 경우 addMarketInfo.html과 mkMarketPg.js와 대/소분류 순서가 일치하는지 신경써야 함
    바뀌게 된다면, 원래 들어 있던 데이터의 의미가 달라질 수 있기 때문에 모든 정보에 대한 업데이트 필요

3. mkMarketPg.js
    옥션의 마켓 탭에 필요한 정보들을 불러오고, 정보 수 만큼 정보를 담은 카드를 만들어 띄움
    아직 로그인 기능이 연결되지 않음, 그 부분은 주석 처리해둠 (31번째 줄, 수정하지 않으면 kimgostring의 데이터가 바뀌게 됨)
    기본 정렬은 최신순(firestore에 정보를 늦게 넣은 순서), 다른 정렬 기준을 선택하더라도 2차 정렬은 항상 최신순임
    관심순은 하트를 사용자가 많이 누른 것부터 띄우게 됨
    판매 기간이 지난 상품/서비스는 페이지에 띄우지 않음
    * 프론트의 경우, 옥션 마켓의 카드 디자인을 바꿀 필요가 있을 때 
    mkCardDom() 함수를 수정하면 됨
    - 카드의 맨 바깥 html 수정하는 경우 : cardContainer.classList에 클래스를 추가/삭제하여 수정
    - 그 외 부분의(안쪽의) html 수정하는 경우 : innerHTML 수정하는 부분의 코드 수정
